Presentation

Access-keys are part of the Web Content Accessibility Guidelines 1.0 (commonly known as WAI "Web Accessibility Initiative", http://www.w3.org/WAI/.) and were introduced to facilitate user navigation allowing him to access parts of an HTML document by typing a succession of keys on his keyboard. Sadly, as Joe Clark says so:

« accesskey is the delinquent teenager of accessible HTML » Joe Clark, "Building Accessible Websites", 2002 (Chapter 8 #h4-3425) - for a partial online version, see http://joeclark.org/book/sashay/serialization/home.html. Joe Clark, "Building Accessible Websites", 2002

The first reason is that all internet browsers are not visual and all internet users are not able to type on a keyboard. But this feature can be useful when you create a full web application and wants to propose quick access to specific common actions to your final users. Anyway, the accesskey's attribute is still part of the HTML5 language See the W3C HTML5 Recommandations : http://www.w3.org/TR/html5/editing.html#the-accesskey-attribute..

The AccessKey plugin for jQuery takes this idea a step further by defining an API to manage, organize and display accesskeys informations on a document model. It enlarges the default usage and mechanism of the HTML accesskey attribute to handle some new actions, defines a unified way to call them, cross-browser and cross-system and allows to totally disable their usage when necessary. Its very first goal is to construct some web pages including the accessibility recommandations, without the matter of browser or system compliance and with a complete control of their usage for final user.

This plugin tries to help you build a universal accessibility in your web pages (since browsers do not, it is up to developers to make this).

Quick usage

The plugin is invoked using the jQuery.fn.slideshow() new function:

<script src="path/to/js/jquery-last.js"></script>
<script src="path/to/js/jquery-slideshow.js"></script>
<link href="path/to/css/jquery-slideshow.css" rel="stylesheet" type="text/css" media="screen" />
<script>

$(function(){ // jQuery on document ready

    // creation of a plugin instance
    $('#my_slideshow').slideshow({
        diaporama_time:     8000,
        container_width:    $(window).width()-80,
        container_height:   $(window).height()-80,
        common_details:     "a detail string for all images, with HTML tags",
        debug:              true
    });

    // define an option after instance creation
    $('#my_slideshow').slideshow('option', 'lang', 'fr');

    // retrieve an option value
    var slideshow_width = $('#my_slideshow').slideshow('option', 'container_width');

});
</script>

<div id="my_slideshow">
    <img src="thumb1.jpg" alt="photo1.jpg" data-title="A custom title, what you want with <em>HTML tags</em>" data-description="And now a description string ... again, what you want ;) like <em>HTML tags</em>" data-details="And a detail : something written very small ... <em>HTML tags</em>" />
    ...
</div>

Demonstration requirements

As you downloaded a package, these HTML pages and all their dependencies are local on your computer or server. If you encounter errors by viewing them or using the plugin, make sure your browser have access to the various javascript files required.
